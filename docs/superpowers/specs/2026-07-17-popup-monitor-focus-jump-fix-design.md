# 弹窗监控误触导致页面跳转修复设计

## 背景

自动截图阶段启动 `PopupMonitor` 后，云游戏会反复跳回腾讯先锋页面。停止程序或禁用异步弹窗监控后现象消失，说明跳转由后台监控的全局鼠标点击触发。

当前监控除两个关闭按钮外，还会以默认模板阈值检测并点击退出确认和通用确认按钮。默认阈值较低，云游戏视频压缩和相似 UI 容易造成误匹配。监控使用 `pyautogui` 全局点击，因此误匹配会直接操作当前前台页面。

## 目标

- 后台监控仅自动点击明确的 X 关闭按钮。
- 后台检测和点击使用相同的屏幕上半区范围。
- 后台监控禁止使用校准坐标兜底，避免模板未匹配时盲点。
- 登录、退出和其他确认按钮继续由主流程在明确步骤中操作。
- 保持现有同步清理、异步扫描和截图流程不变。

## 方案

### Navigator

为 `wait_for_template` 增加可选 `bounds` 参数，使模板检测可以限制在指定区域。

为 `find_and_click` 增加可选 `allow_fallback` 参数，默认值为 `True`，保持现有调用兼容。传入 `False` 时，模板匹配失败直接返回 `False`，不读取或点击 `calibrated_coords.json` 中的坐标。

### PopupMonitor

监控按钮白名单仅保留：

- `popup_close.png`
- `popup_close_small.png`

两者均使用 `0.85` 阈值，并在检测与点击时传入相同的上半屏 `bounds`。调用 `find_and_click` 时传入 `allow_fallback=False`。

删除后台监控中的 `game_logout_confirm.png` 和 `game_popup_confirm.png`。这些按钮仍可由登录、退出等主流程显式调用，不改变相应业务行为。

## 错误处理

模板不存在、匹配失败或截图异常时，监控记录日志并跳过本轮，不执行校准坐标或屏幕空白区域兜底点击。只有成功匹配白名单中的 X 按钮时才允许在该按钮位置产生鼠标事件。

## 测试

- 验证 `wait_for_template` 只在指定范围内匹配。
- 验证 `find_and_click(..., allow_fallback=False)` 匹配失败时不会执行校准坐标点击。
- 验证 `PopupMonitor` 只查询两个 X 按钮，并向检测和点击传递上半屏范围及禁用兜底参数。
- 运行现有测试，确认登录、导航和截图行为没有回归。

## 验收标准

- 启动异步弹窗监控后，手动切回云游戏不会再被监控点击带回腾讯先锋页面。
- 后台监控日志中不再出现退出确认或通用确认模板的检测、点击。
- 真正出现已配置的 X 关闭按钮时仍能自动关闭。
- 主流程中的游戏退出确认仍能正常执行。
