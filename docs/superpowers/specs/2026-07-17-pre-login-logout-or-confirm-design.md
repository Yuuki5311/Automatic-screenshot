# 游戏登录前：退出按钮或确认弹窗处理

## 背景

进入阶段 3 游戏登录前，需先清掉自动登录残留或云服务器确认弹窗。
原先未检测到退出按钮时直接跳过；有退出时点击的是 `game_logout_confirm.png`。

## 目标

首次启动（阶段 2 末）与「再跑一轮」两处统一为：

1. 同步清理 X 关闭按钮（现有逻辑不变）。
2. 若检测到 `game_logout_btn.png`：点击退出 → 再点下半屏 `game_popup_confirm.png`。
3. 若未检测到退出按钮：在下半屏检索 `game_popup_confirm.png`；有则点击，无则跳过。
4. 然后继续阶段 3 游戏登录。

## 非目标

- 不改异步 `PopupMonitor` 白名单（仍不自动点确认类按钮）。
- 不改阶段 4 截图后的设置退出流程。
- 不删除 `game_logout_confirm.png` 模板文件（仅本路径不再使用）。

## 实现位置

`gui/app.py` 中两处相同的退出按钮检测分支（首次启动与再跑一轮）。

确认点击使用与现有一致的下半屏 bounds：
`(0, BROWSER_HEIGHT // 2, BROWSER_WIDTH, BROWSER_HEIGHT // 2)`。

## 验收

- 有退出按钮时日志依次出现：已点击退出登录 → 已确认（`game_popup_confirm`）。
- 无退出按钮但有确认弹窗时：点击确认后进入阶段 3。
- 两者皆无时：跳过并继续阶段 3，不中断工作流。
