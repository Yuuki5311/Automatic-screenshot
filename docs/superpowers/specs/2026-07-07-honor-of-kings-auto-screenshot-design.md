# 王者荣耀云游戏自动截图脚本 — 设计文档

## 概述

通过 Python 脚本自动登录腾讯先锋云游戏中的王者荣耀，并对指定游戏页面进行截图保存。

## 运行环境

- **操作系统**：macOS (Darwin 24.x)
- **屏幕分辨率**：2560 × 1664 Retina
- **浏览器**：Chrome + ChromeDriver
- **Python**：3.10+

## 核心技术

| 层 | 工具 | 用途 |
|---|---|---|
| 浏览器自动化 | Selenium WebDriver | 打开腾讯先锋网页、QQ登录、启动游戏 |
| 游戏内交互 | OpenCV 模板匹配 | 识别游戏画面中的按钮/图标位置，模拟点击 |
| 截图处理 | Pillow + Selenium screenshot | 截取当前画面并按名称保存 |

### 为什么用图像识别

云游戏画面是视频流，无 DOM 可查。所有游戏内按钮定位依赖 `cv2.matchTemplate()` 进行模板匹配。模板图为事先截取的小图，存放在 `templates/` 目录。

### 分辨率适配

Selenium 设置固定浏览器窗口尺寸，所有模板图基于该尺寸截取，保证匹配一致性。

## 模块设计

### main.py — 流程编排

- `run()` 函数按顺序调用各模块
- 每步截图之间加等待（1-2 秒）
- 关键失败（登录失败、游戏启动失败）直接终止
- 非关键失败（单个页面截图失败）记录日志并继续

### login.py — QQ 登录

- 打开腾讯先锋云游戏网址（待确认具体 URL）
- 点击"登录"按钮 → 跳转 QQ 登录页
- 输入 QQ 号（从配置文件读取）
- 输入密码（运行时通过 `getpass` 交互输入）
- 处理可能的验证码（先跳过，遇到再处理）

### game_launcher.py — 启动游戏

- 在腾讯先锋页面中找到王者荣耀
- 点击"开始游戏"
- 等待游戏画面加载完成

### navigator.py — 游戏内导航

核心函数：
- `find_and_click(template_path, timeout=10)` — 在当前画面匹配模板并点击
- `wait_for_template(template_path, timeout=15)` — 等待模板出现（用于确认页面加载）
- `go_back()` — 返回上一级 / 关闭弹窗

匹配策略：
- 每次最多重试 3 次，间隔 2 秒
- 匹配阈值 ≥ 0.8 视为找到
- 连续 3 次失败则跳过并记录警告

### screenshotter.py — 截图

- `take_screenshot(name)` — 截取当前浏览器画面，保存为 `screenshots/{name}.png`
- 使用 Selenium 的 `driver.get_screenshot_as_png()` 获取原始画面

### config.py — 配置

```python
# 腾讯先锋
CLOUD_GAMING_URL = "https://start.qq.com/"  # 待确认

# 浏览器
BROWSER_WIDTH = 1920
BROWSER_HEIGHT = 1080

# 等待时间（秒）
PAGE_LOAD_WAIT = 3
CLICK_INTERVAL = 1.5
TEMPLATE_MATCH_TIMEOUT = 10

# 路径
TEMPLATES_DIR = "templates"
SCREENSHOTS_DIR = "screenshots"
```

## 安全

- QQ 密码通过 `getpass` 在运行时输入，不写入文件
- QQ 号存储在配置文件（不含密码）
- 建议 `.gitignore` 中排除 `config.py`（含敏感信息）和 `screenshots/`

## 已知限制与后续迭代

1. **模板图需首次手动截取**：首次使用需要进入游戏手动截取按钮模板图
2. **验证码处理**：QQ 登录可能触发验证码，当前方案先跳过，遇到后再补充
3. **腾讯先锋页面更新**：平台页面结构变化需要更新 Selenium 选择器
4. **仅支持 Chrome**：不跨浏览器
