# 截图任务「按键」插入设计

## 背景

阶段 4 感知环按 `screenshot_tasks` 有序截图。需在「万象图鉴-灵宝」截图并返回之后，增加一条任务：进入局内相关入口 → 打开按键页 → 截图 → 返回，再继续原有的「天幕」等后续任务。

## 目标

- 在任务表「万象图鉴-灵宝」与「天幕」之间插入任务「按键」。
- 导航：点击「局内」模板 → 点击「按键」模板 → 保存 `按键.png` → 返回 1 次。
- 感知环、登录、启游戏逻辑不变。

## 非目标

- 不抽离公共任务表模块。
- 不增加坐标兜底。
- 不在仓库内伪造真实游戏模板像素（PNG 由本地 `capture_templates.py` 截取）。

## 任务定义

| 字段 | 值 |
|------|-----|
| 名称 / 输出文件 | `按键` → `screenshots/{账号}/按键.png` |
| 点击 1 | `in_game_btn.png`，说明：点击局内按钮 |
| 点击 2 | `keybind_btn.png`，说明：点击按键按钮 |
| `back_count` | `1` |
| bounds | 无（全屏匹配） |

插入后局部顺序：

```
万象图鉴首页 → 万象图鉴-灵宝（截图 → 返回）
  → 按键（局内 → 按键 → 截图 → 返回）
  → 天幕 → …
```

**前提假设：** 灵宝返回后当前页上，「局内」「按键」两个按钮均可见。若实战发现需先导航到其他页，另开变更补点击步。

## 实现改动

| 文件 | 改动 |
|------|------|
| `gui/app.py` | `screenshot_tasks` 在灵宝后插入上述三元组 |
| `test_core.py` | `TestScreenshotTasks` 同步插入，保证结构断言覆盖 |
| `capture_templates.py` | `TEMPLATES` 登记两张新模板及所在页面说明 |

## 模板截取

```text
python capture_templates.py --only in_game_btn.png keybind_btn.png
```

在「灵宝返回后、点局内前」的画面框选保存。未放置真实 PNG 前，阶段 4 跑到该任务会匹配失败并按现有逻辑告警/跳过推进（与其它缺模板行为一致）。

## 测试

- 单元：`TestScreenshotTasks` 含「按键」任务，结构与 `.png` 后缀合法。
- 实战：模板就位后，灵宝 → 按键 → 天幕顺序正确，输出 `按键.png`。

## 风险

| 风险 | 对策 |
|------|------|
| 灵宝返回后不在含「局内」的页 | 补导航点击或调整插入点 |
| 模板过小/过大导致误匹配 | 用 capture 工具重截；必要时加 bounds |
